What it does

Daily Market

Per-contract end-of-day snapshot: listed markets, open interest, daily volume, high/low. Breadth (listed vs. traded), mix by sport / market / period.

Daily Settlement

How each contract resolved — $1 (YES) / $0 (NO) / $0.50 (push). Also the source of verified team/player names (Railbird's own labels).

Time & Sales

The trade tape: every execution with its intraday price (= implied probability), timing and size. VWAP, price paths, activity by hour.

MLB KBO NPB PGA — new leagues (soccer / NFL / NBA) load automatically and are flagged by the built-in new-sport detector.

Read the numbers correctly

Contracts ≠ dollars. Trade volume counts binary contracts that settle at $1 or $0. Dollar figures are estimates, always labeled: Notional @ price = Σ(volume × last price); Max notional = Σ(volume × $1) ceiling.
  • Verified names: team/player names come from the settlement report where available, else a heuristic fallback, else the raw code.
  • “Distinct events” is approximate — the event-id token can differ across market types for one real-world game.
  • Railbird ≠ all of DKEX — these reports cover only the regulated-exchange layer, not DraftKings' full consumer / market-making flow.

Run it locally

git clone https://github.com/nosherzapoo/dkex-railbird-dashboard.git
cd dkex-railbird-dashboard
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
streamlit run app/dashboard.py   # builds its database from bundled CSVs on first run

To refresh data: python -m dkex.fetch then python -m dkex.load.
